Treap (Árvore + Heap)

Estrutura de dados que combina as propriedades de uma Árvore de Busca Binária e de um Heap.

O que é uma Treap

Uma Treap é uma estrutura de dados que combina as propriedades de uma Árvore Binária de Busca (BST) e um Heap. Cada nó tem um valor (seguindo a propriedade BST) e uma prioridade aleatória (seguindo a propriedade Max-Heap).

Formalmente, um Treap é definido como uma árvore binária na qual cada nó contém:

  • Key (Chave):Valor que determina a posição do nó segundo a ordenação BST;
  • Priority (Prioridade): Valor numérico, geralmente sorteado de forma uniforme e independente, que determina a posição do nó segundo as regras de heap;
  • Balanceamento (Left and Right): Ponteiros para as subárvores esquerda e direita.

Algumas aplicações são:

    Características:

    • Valor: Segue a propriedade de Árvores Binárias de Busca (BST) (esquerda < raiz < direita)
    • Prioridade: Segue a propriedade de Max-Heap (pai > filhos)
    • Balanceamento: Prioridades aleatórias garantem balanceamento esperado
    • Operações: O(log n) esperado para inserção, busca e remoção

    Comparação: Treap vs. AVL vs. Red-Black

    Operações Split e Merge

    Operações fundamentais que aproveitam a estrutura do Treap

    Split

    Divide a Treap em duas árvores baseado em um valor chave. Todos os elementos menores que a chave vão para a árvore da esquerda, e os maiores para a direita.

    split(treap, key) →
      {{ '{' }} left_tree, right_tree {{ '}' }}
    • • Complexidade: O(log n) esperado
    • • Útil para consultas de range

    Merge

    Combina duas Treaps em uma única árvore. Assume que todos os elementos da primeira árvore são menores que os da segunda.

    merge(left_tree, right_tree) →
      combined_tree
    • • Complexidade: O(log n) esperado
    • • Mantém propriedades do heap
